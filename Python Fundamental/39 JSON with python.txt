### Part 1: Theory of JSON with Python

#### What is JSON in Python?
JSON (JavaScript Object Notation) is a lightweight, text-based data format used to store and exchange structured data. It is human-readable, language-independent, and widely used in web APIs, configuration files, and data storage. In Python, the `json` module provides built-in functionality to parse (read) and serialize (write) JSON data. In 2025, JSON remains a cornerstone for data interchange in web development (FastAPI, Django), data science, and automation due to its simplicity and compatibility with modern frameworks.

#### Key Components
1. **JSON Structure**:
   - JSON supports data types: objects (`{}`), arrays (`[]`), strings, numbers, booleans, and null.
   - Objects are key-value pairs (like Python dictionaries).
   - Arrays are ordered lists (like Python lists).
   - Example: `{"name": "Alice", "age": 25, "hobbies": ["reading", "coding"]}`.

2. **Python’s `json` Module**:
   - **Serialization**: Convert Python objects (e.g., dict, list) to JSON strings using `json.dump()` or `json.dumps()`.
   - **Deserialization**: Parse JSON strings into Python objects using `json.load()` or `json.loads()`.
   - Example: `json.dumps({"name": "Bob"})` produces a JSON string.

3. **File Operations**:
   - **Writing JSON**: Use `json.dump()` to write to a file.
   - **Reading JSON**: Use `json.load()` to read from a file.
   - Example: `with open('data.json', 'w') as f: json.dump(data, f)`.

4. **Exception Handling**:
   - Handle errors like `json.JSONDecodeError` for invalid JSON or `FileNotFoundError` for missing files.
   - Example: `try: json.load(f) except json.JSONDecodeError: print("Invalid JSON")`.

5. **Encoding**:
   - JSON uses UTF-8 encoding by default in Python.
   - Specify encoding when reading/writing files (e.g., `open('data.json', encoding='utf-8')`).

#### How JSON Processing Works
- **Serialization (`json.dumps`, `json.dump`)**:
  - Converts Python objects to JSON strings or files.
  - Supports basic types: `dict`, `list`, `str`, `int`, `float`, `bool`, `None`.
  - Custom objects require a custom encoder (e.g., subclass `json.JSONEncoder`).
  - Example: `json.dumps({"key": "value"})` → `"{\"key\": \"value\"}"`.
- **Deserialization (`json.loads`, `json.load`)**:
  - Parses JSON strings or files into Python objects.
  - JSON objects become `dict`, arrays become `list`, etc.
  - Example: `json.loads('{"key": "value"}')` → `{"key": "value"}`.
- **Performance**:
  - **Time Complexity**: O(n) for serializing/deserializing n characters.
  - **Space Complexity**: O(n) for storing JSON data in memory.
- **File I/O**:
  - JSON file operations use Python’s file handling (`open()` with `with` statements).
  - Example: `json.dump(data, open('data.json', 'w'))`.

#### Key Features
- **Simplicity**: JSON’s syntax is easy to read and write.
- **Interoperability**: Compatible with most programming languages and APIs.
- **Flexibility**: Handles nested structures (objects, arrays).
- **Integration**: Works seamlessly with `pandas`, `FastAPI`, `Django`, and `Pydantic`.
- **Portability**: JSON files are platform-independent and lightweight.

#### Use Cases
- **Web Development**:
  - Exchange data in APIs (FastAPI, Django REST Framework).
  - Example: Return JSON responses in FastAPI: `return {"status": "success"}`.
- **Data Science**:
  - Store and load datasets or configurations with `pandas`.
  - Example: `df.to_json('data.json')`.
- **Automation**:
  - Save script configurations or outputs.
  - Example: Store settings in `config.json`.
- **Machine Learning**:
  - Save model metadata or hyperparameters.
  - Example: Save training config in `model_config.json`.
- **Frameworks in 2025**:
  - **FastAPI**: Serialize/deserialize JSON for API requests/responses.
  - **Django**: Use `JsonResponse` or serialize models to JSON.
  - **Streamlit**: Load JSON configs for dashboards.
  - **Pydantic**: Validate JSON data with schema enforcement.

#### Advanced Concepts
- **Custom Serialization**:
  - Use a custom `json.JSONEncoder` for non-serializable objects (e.g., `datetime`).
  - Example:
    ```python
    import json
    class CustomEncoder(json.JSONEncoder):
        def default(self, obj):
            if isinstance(obj, datetime):
                return obj.isoformat()
            return super().default(obj)
    ```
- **Custom Deserialization**:
  - Use `object_hook` or `object_pairs_hook` in `json.loads()` to customize parsing.
  - Example: Convert JSON keys to Python objects.
- **Async JSON Handling**:
  - Use `aiofiles` with `json` for async file operations in FastAPI.
  - Example:
    ```python
    import aiofiles
    async with aiofiles.open('data.json') as f:
        data = json.loads(await f.read())
    ```
- **Validation**:
  - Use `Pydantic` to validate JSON data against a schema.
  - Example: Define a `Pydantic` model to enforce JSON structure.
- **Security**:
  - Avoid loading untrusted JSON to prevent injection attacks.
  - Validate JSON schemas with `jsonschema` or `Pydantic`.
- **Pretty Printing**:
  - Use `indent` in `json.dump()`/`json.dumps()` for readable output.
  - Example: `json.dumps(data, indent=2)`.

#### Best Practices
- Always use `with` statements for file operations to ensure proper closure.
- Handle `json.JSONDecodeError` for invalid JSON data.
- Specify `encoding='utf-8'` for JSON file operations.
- Use `pathlib` for cross-platform file handling.
- Validate JSON data with `Pydantic` or `jsonschema` in production.
- Use `indent` for human-readable JSON output during debugging.
- In 2025, integrate with `Pydantic` for schema validation or `aiofiles` for async apps.
- Test JSON operations with `pytest` using temporary files.

#### Layman’s Explanation of JSON with Python
JSON in Python is like using a structured notebook to store and share information. It’s a way to write data (like names, numbers, or lists) in a format that both humans and computers can understand. You can save this notebook to a file (`json.dump`) or read it back (`json.load`). Python’s `json` module is like a librarian who translates between Python’s language (dictionaries, lists) and JSON’s format. You need to be careful about errors, like a poorly formatted notebook, and use tools like `Pydantic` to ensure the data makes sense.

#### Question 1
**Question**: What is JSON, and how is it used in Python?  
**Expected Answer**: JSON is a lightweight data format for storing and exchanging data. Python’s `json` module handles serialization (`dump`, `dumps`) and deserialization (`load`, `loads`).  
**Sample Code**:
```python
import json
data = {"name": "Alice"}

```
**Layman’s Explanation**: JSON is like a structured notebook; Python’s `json` module helps you write and read it.

#### Question 2
**Question**: What is the difference between `json.dump()` and `json.dumps()`?  
**Expected Answer**: `json.dump()` writes JSON to a file; `json.dumps()` returns a JSON string.  
**Sample Code**:
```python
import json
data = {"name": "Bob"}
with open('data.json', 'w') as f:
    json.dump(data, f)
json_str = json.dumps(data)
print(json_str)  # Output: {"name": "Bob"}
```
**Layman’s Explanation**: `dump` saves the notebook to a file; `dumps` gives you the notebook’s contents as text.

#### Question 3
**Question**: How do you read JSON from a file in Python?  
**Expected Answer**: Use `json.load()` to read JSON from a file into a Python object.  
**Sample Code**:
```python
import json
with open('data.json', 'w') as f:
    json.dump({"name": "Alice"}, f)
with open('data.json', 'r') as f:
    data = json.load(f)
    print(data['name'])  # Output: Alice
```

#### Question 11
**Question**: How do you handle non-serializable objects in JSON?  
**Expected Answer**: Use a custom `json.JSONEncoder` to serialize objects like `datetime`.  
**Sample Code**:
```python
import json
from datetime import datetime
class CustomEncoder(json.JSONEncoder):
    def default(self, obj):
        if isinstance(obj, datetime):
            return obj.isoformat()
        return super().default(obj)
data = {"date": datetime(2025, 1, 1)}
print(json.dumps(data, cls=CustomEncoder))  # Output: {"date": "2025-01-01T00:00:00"}
```
**Layman’s Explanation**: Custom encoding is like translating special notes (like dates) into a JSON-friendly format.

#### Question 19
**Question**: How do you convert JSON to a Python class object?  
**Expected Answer**: Use `object_hook` in `json.loads()` or `Pydantic` to map JSON to objects.  
**Sample Code**:
```python
import json
def as_object(dct):
    class Obj: pass
    obj = Obj()
    obj.__dict__.update(dct)
    return obj
data = json.loads('{"name": "Alice"}', object_hook=as_object)
print(data.name)  # Output: Alice
```
**Layman’s Explanation**: Converting JSON to objects is like turning a notebook’s notes into a structured Python object.
